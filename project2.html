    End Users

    Our end users are primarily children. This is because Tic Tac Toe is more of a fun, quick-paced and quick game. We will be designing it specifically for users aged 8-16, so this means the language will need to be simple enough for everyone to understand.

    Requirements

  • Singeplayer
    • The player always starts first in the game, not the computer.
    • It will have a computer that will play random / best moves against the player.
    • Asks how many rounds they want to play.
    • Scoring system
    • +5 score for the player that wins.
    • Score telling at end
    • Asks them if they want to replay.
  • Multiplayer
    • Question that asks both players if they want to start.
    • If both players want to start, 50/50 chance.
    • If only one player wishes to start, they will.
    • If no players want to start, 50/50 chance.
    • Asks how many rounds they want to play.
    • Scoring system
    • +5 score for the player that wins.
    • Score telling at end
    • Asks them if they want to replay.
  • Introduction
    • Asks if they want Singeplayer or Multiplayer
    • Asks for the users names.
  • Instructions
    • Explains the game in detail and clearly so the user can understand.
    • Informs the user on how to use the game (controls).

    Implications

    Functionality

    Within the game of Tic Tac Toe, functionality is ensuring that the game itself actually works. This means that everything should work as intended and no errors occur in the code.

    How I will address this:

  • I will address this implication by playing through the game multiple times and testing every part of the game to ensure it is functional.
  • I can also double check the functionality by reading the code and seeing if there are any errors that need to be fixed.

    • Usability

    The usability of a Tic Tac Toe game is ensuring that the game is easy and simple to use. We need to ensure that our users can easily understand how the game works and that it is not too complicated to use.

    How I will address this:

  • I will get feedback from people in my class on how easy it is to use.
  • I will also give users the instructions so they can read them and see if they're easy to understand.

    • Privacy

    Privacy is all about keeping a users personal details that they woudln't want being shared safe. This means their name, phone, card, etc. For our game, it would mostly just be their name that needs to be kept safe.

    How I addressed them:

  • We will ensure the privacy of the users name as it is not stored in a computer, instead it is in a variable that is deleted once the application is closed.
